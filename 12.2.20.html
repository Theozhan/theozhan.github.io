How to use Git to post your first project?

12 Feb, 2020
Git

Git is a distributed version control system where each user can make changes to an entire repository on their system. It uses command line and with git it’s easy to undo changes back and forth with a precise explanation of the changes that are made.

It has the ability of:

  • Keeping a track of the changes, like different versions of the same file.
  • It also keeps a record of all the files present in a project.
  • Comparing and analyzing different codes with a detailed explanation.
Installing Git

Before you start using Git, you have to make it available on your computer. Even if it’s already installed, it’s probably a good idea to update to the latest version. You can either install it as a package or via another installer, or download the source code and compile it yourself.

Git for Windows

There are also a few ways to install Git on Windows. The most official build is available for download on the Git website. Just go to https://git-scm.com/download/win and the download will start automatically. Note that this is a project called Git for Windows, which is separate from Git itself; for more information on it, go to https://gitforwindows.org

Another easy way to get Git installed is by installing GitHub Desktop. The installer includes a command line version of Git as well as the GUI. It also works well with PowerShell, and sets up solid credential caching and sane CRLF settings.

How to push my file to the master branch?
  • git init initial your local file for the git respository
  • add . add your file to your local folder (respository)
  • git commit -m "First commit" add the commit otherwise it won't show the file
  • git remote add origin https://github.com/YOUR RESPOSITORY add your local folder to the remote respository by using URL
  • git push origin master replace your remote respository file to your new local folder by using the push command
Some Questions
  • Why said "everything up-to-date" but no files on Gitlab/Github?

    • A: You didn't add the commit, try git commit -m 'first commit' after pulling the master.

  • Why said "error:src refspec master does not match any after pushing"

    • because there's no file under the master branch, should add the README file instead. git add README and git commit -m 'first commit'
